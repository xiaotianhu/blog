Hyperf源码阅读 - 框架初始化之Application

Application的入口在bin/hyperf.php中:

$application = $container->get(\Hyperf\Contract\ApplicationInterface::class);
$application->run();

拿到Application,然后run().
找到ApplicationInterface的绑定关系,在

vendor/hyperf/framework/src/ConfigProvider.php中

ApplicationInterface::class => ApplicationFactory::class

核心实现在ApplicationFactory了.来看工厂类

__invoke()
先dispatch了一个BootApplication的Event.看一下这个Event在vendor/hyperf/framework/src/Event目录下面,这里定义的是框架的各个事件.

通过Config管理对象获取全部的$commands,这个commands就是框架支持的一些命令,或者用户自己定义的一些命令.
后面继续加载通过注解annotation定义的commands

\(application对象是Symfony\Component\Console\Application,直接new生成对象,然后把SymfonyEventDispatcher注入进application依赖.
每一个命令字符串,通过\)Container拿到最终的对象,通过add方法加入到application里面,工厂的任务就结束了
最终把依赖都注入到$application后返回

$application->run()
start命令执行的入口.
先设置了一些基本的环境,Excenption处理的Handler之类.

  • configureIO()
    解析基本的输入命令,设置报错等级
  • doRun()
    执行命令 套在一个try catch finally里面,最上层的catch,同时在finally里面做一些清理

运行成功会返回0;失败返回error code.

Application->doRun()
执行命令前 也是先dispatch一些Event
最终调用$command->run()方法
如果有Throwable的错误在catch一下
执行后dispatch一个TerminalEvent

StartServer.php
启动时输入的命令 start 最终执行的类在
vendor/hyperf/server/src/Command/StartServer.php

先拿到Hyperf\Server\ServerFactory
注入EventDispatcher和Logger依赖
